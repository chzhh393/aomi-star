## 📋 背景

经纪人目前在管理后台（admin-web）进行面试打分和上传资料，但这些操作在手机上更方便：
- **面试现场**：经纪人需要随时随地查看候选人信息、快速打分
- **拍照录视频**：手机上操作更自然、更快捷
- **移动办公**：经纪人经常外出，需要移动端支持

同时，经纪人需要能够方便地分享候选人信息给同事或合作伙伴，但需要确保敏感信息不被泄露。

## 🔨 实现方案

### 技术方案

#### 1. 认证系统
- 创建独立的 `agent-auth.js` 处理用户名密码登录（区别于微信登录）
- Token 本地存储和自动过期处理
- 统一的错误处理和跳转机制

#### 2. 数据脱敏
- 云函数已实现 ROLE_PERMISSIONS 配置
- 经纪人角色自动过滤敏感字段：phone、wechat、referral
- 前端二次验证确保数据安全

#### 3. 文件上传优化
- 先显示临时图片，提升用户体验
- 后台异步上传到云存储，带进度显示
- 批量上传支持，自动重试机制

#### 4. 分享功能
- 使用微信小程序 `onShareAppMessage` API
- 分享数据仅包含基本信息（姓名、年龄、身高、城市）
- 使用候选人照片作为分享封面

## 🐛 遇到的问题

### 问题1: substr() 方法 deprecated 警告

**原因**: 使用了已废弃的 `String.prototype.substr()` 方法

**解决方案**:
```javascript
// 修改前
const randomStr = Math.random().toString(36).substr(2, 9);

// 修改后
const randomStr = Math.random().toString(36).substring(2, 11);
```

### 问题2: 底部操作栏按钮过多导致布局拥挤

**原因**: 添加分享按钮后，底部有3个按钮，空间紧张

**解决方案**:
- 保持 `flex: 1` 布局，三个按钮平分空间
- 使用橙色渐变区分分享按钮
- 添加分享图标提升识别度
