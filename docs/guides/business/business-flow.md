## 阶段0：用户注册和角色识别

### 🎯 阶段目标
用户首次进入小程序时,根据不同入口和身份快速完成注册并路由到对应工作台。

### 📋 三种入口场景

#### 场景1：星探推荐码进入

```mermaid
sequenceDiagram
    participant C as 候选人
    participant MP as 小程序
    participant CF as 云函数
    participant DB as 数据库

    C->>MP: 扫描星探推荐二维码
    MP->>MP: 解析推荐码SC-EXT-xxx
    MP->>C: 显示授权页面
    C->>MP: 同意微信授权
    MP->>CF: 调用login(推荐码)
    CF->>DB: 查询用户(openid)

    alt 用户不存在
    CF-->>MP: user=null
    MP->>C: 跳转报名页(自动填充推荐码)
    C->>MP: 填写报名表单
    MP->>CF: 调用createCandidate
    CF->>DB: 创建用户(role=candidate)
    CF->>DB: 创建推荐记录
    CF-->>MP: 注册成功
    MP->>C: 跳转候选人工作台
    else 用户已存在
    CF-->>MP: 返回用户信息
    MP->>C: 根据role跳转对应工作台
    end
```

**候选人体验**：
1. 扫码后看到"XX星探推荐您应聘主播"
2. 微信授权登录
3. 填写报名表(推荐码自动填充,不可见)
4. 提交后立即进入候选人工作台,可查看进度

---

#### 场景2：员工邀请码进入

```mermaid
sequenceDiagram
    participant E as 新员工
    participant MP as 小程序
    participant CF as 云函数
    participant DB as 数据库

    E->>MP: 扫描HR邀请二维码
    MP->>MP: 解析邀请码INV-HR-xxx
    MP->>E: 显示授权页面
    E->>MP: 同意微信授权
    MP->>CF: 调用verifyInviteCode
    CF->>DB: 验证邀请码有效性

    alt 邀请码有效
    CF-->>MP: 返回角色信息
    MP->>E: 显示信息绑定页
    E->>MP: 输入手机号+姓名
    MP->>CF: 调用bindEmployee
    CF->>DB: 验证手机号在员工预登记表
    CF->>DB: 绑定openid到员工记录
    CF->>DB: 创建用户(role=employee)
    CF-->>MP: 绑定成功
    MP->>E: 跳转员工工作台
    else 邀请码无效
    CF-->>MP: 邀请码无效或已过期
    MP->>E: 提示联系HR
    end
```

**员工体验**：
1. 扫描HR发送的邀请码
2. 输入手机号和姓名验证身份
3. 绑定成功后直接进入对应角色工作台

---

#### 场景3：直接搜索进入

```mermaid
sequenceDiagram
    participant U as 用户
    participant MP as 小程序
    participant CF as 云函数
    participant DB as 数据库

    U->>MP: 搜索小程序名称打开
    MP->>U: 显示授权页面
    U->>MP: 同意微信授权
    MP->>CF: 调用login
    CF->>DB: 查询用户(openid)

    alt 用户已注册
    CF-->>MP: 返回用户信息
    MP->>U: 根据role跳转工作台
    else 用户未注册
    CF-->>MP: user=null
    MP->>U: 显示身份选择页

    alt 选择应聘主播
    U->>MP: 点击"我要应聘主播"
    MP->>U: 跳转报名页
    U->>MP: 填写报名表单
    MP->>CF: createCandidate
    CF->>DB: 创建用户(role=candidate)
    MP->>U: 跳转候选人工作台

    else 选择成为星探
    U->>MP: 点击"我是星探"
    MP->>U: 跳转星探注册页
    U->>MP: 填写星探信息
    MP->>CF: createScout
    CF->>DB: 创建用户(role=external_scout)
    MP->>U: 跳转星探工作台

    else 选择公司员工
    U->>MP: 点击"我是公司员工"
    MP->>U: 提示需要邀请码
    U->>MP: 输入邀请码(同场景2)
    end
    end
```

**用户体验**：
1. 搜索小程序打开
2. 如果已注册,直接进入对应工作台
3. 如果未注册,选择身份后完成注册流程

---

### 🎯 关键设计要点

1. **候选人即时注册**：
   - ✅ 候选人报名时立即创建账号(role=candidate)
   - ✅ 不需要等到签约才绑定微信
   - ✅ 注册后可实时查看申请进度

2. **角色唯一性**：
   - ✅ 一个微信只能注册一个角色
   - ✅ 重复注册会提示已有身份

3. **推荐码追踪**：
   - ✅ 星探推荐码自动记录在candidateInfo中
   - ✅ 用于后续佣金结算

4. **员工邀请验证**：
   - ✅ 员工需通过手机号+姓名验证
   - ✅ 确保只有预登记的员工才能绑定

## 阶段1：候选人报名

### 🎯 阶段目标
候选人通过小程序完成微信登录并提交报名信息，系统创建候选人账号。

### 📋 流程详解

```mermaid
sequenceDiagram
    participant C as 候选人
    participant MP as 小程序
    participant CF as 云函数
    participant DB as 数据库

    C->>MP: 1. 进入小程序
    MP->>C: 2. 请求微信授权
    C->>MP: 3. 同意授权
    MP->>CF: 4. 调用login云函数
    CF->>DB: 5. 创建账号(role=candidate)
    CF->>MP: 6. 返回用户信息
    C->>MP: 7. 填写报名表单
    C->>MP: 8. 上传照片和视频
    MP->>CF: 9. 提交报名信息
    CF->>DB: 10. 保存candidateInfo
    CF->>DB: 11. 更新status=pending
    CF->>MP: 12. 返回成功
    MP->>C: 13. 显示提交成功页
```

### 📝 信息采集清单

#### 1. 基础信息（必填）
- 真实姓名、艺名
- 年龄、性别
- 身高、体重
- 联系方式（手机、微信）
- 所在地（省市区）
- 身份证号（加密存储）

#### 2. 形象资料（必填）
- 生活照 3张
  - 正面照（五官清晰）
  - 侧面照（展示轮廓）
  - 全身照（展示身材比例）
- 艺术照 3张（不同风格）
- 自我介绍视频 1分钟
- **系统自动生成**：AI颜值评分报告

#### 3. 才艺信息
- 主修才艺（多选）：唱歌、舞蹈、乐器、脱口秀、其他
- 才艺等级：1-10级自评
- 代表作品链接（如有）
- 特殊才艺说明

#### 4. 经验信息
- 直播经验：有/无
- 曾用平台：抖音、快手、B站等（多选）
- 历史最高粉丝数
- 历史最高月收入
- 代表性内容链接

#### 5. 性格特质
- MBTI人格类型
- 抗压能力自评：1-5分
- 团队协作意愿：1-5分
- 职业目标描述（文本）

#### 6. 期望条件
- 期望薪资范围：底薪区间
- 可投入时间：全职/兼职，每周时长
- 内容方向偏好：舞蹈、才艺、聊天、游戏等
- 其他特殊要求

### 🔄 状态变化
- **输入状态**：无（新用户）
- **输出状态**：`pending`（待审核）

### 📊 数据记录
- ✅ 创建users集合记录（role=candidate）
- ✅ 创建candidateInfo子文档
- ✅ 记录推荐码（如来自星探推荐）
- ✅ 记录提交时间和渠道来源

### 👥 涉及角色
- **候选人**：填写报名表单
- **系统**：创建账号，保存数据

### 👤 候选人视角

#### 操作步骤
1. **进入小程序**：通过星探推荐码/员工邀请/直接搜索进入
2. **微信授权**：同意授权获取微信信息，系统自动创建账号
3. **填写报名表**：完成6大类信息采集(约10-15分钟)
4. **上传素材**：照片6张+视频1个
5. **提交审核**：点击提交，状态变为"待审核"

#### 看到的界面
- **报名进度条**：实时显示填写进度(0%-100%)
- **提交成功页**：
  - "您的简历已提交成功!"
  - "HR将在1-2个工作日内审核"
  - "审核结果将通过微信消息通知您"
  - [进入我的工作台] 按钮

#### 进入候选人工作台后
- **申请状态**：显示"待审核" (status=pending)
- **提交时间**：2025-01-05 14:30
- **可操作**：
  - 查看完整简历
  - 修改简历(审核前可修改)

#### 收到的通知
- **微信模板消息**：
  - "您的主播应聘简历已提交"
  - "预计1-2个工作日内完成审核"
  - 点击查看详情→候选人工作台

## 阶段2：HR审核与面试官分配

### 🎯 阶段目标
HR审核候选人简历，筛选合格者并分配面试官团队。

### 📋 流程详解

```mermaid
sequenceDiagram
    participant HR as HR管理员
    participant S as 系统
    participant DB as 数据库
    participant C as 候选人

    HR->>S: 1. 进入候选人池
    S->>DB: 2. 查询pending状态候选人
    DB->>S: 3. 返回候选人列表
    S->>HR: 4. 展示候选人信息
    HR->>HR: 5. 审核简历和素材

    alt 审核通过
        HR->>S: 6a. 点击"通过"
        HR->>S: 7a. 分配面试官
        Note over HR: 选择舞蹈导师、经纪人<br/>化妆师、造型师
        S->>DB: 8a. 更新status=interview_scheduled
        S->>DB: 9a. 保存面试官分配
        S->>C: 10a. 发送面试通知
    else 审核拒绝
        HR->>S: 6b. 点击"拒绝"
        HR->>S: 7b. 填写拒绝原因(可选)
        S->>DB: 8b. 更新status=rejected
        S->>C: 9b. 发送拒绝通知
    end
```

### 📋 HR审核要点

#### 审核重点
1. **基本条件检查**
   - 年龄、身高等基本要求
   - 信息完整性

2. **形象评估**
   - 照片真实性和质量
   - 镜头表现力
   - 整体气质

3. **素材质量**
   - 自我介绍视频流畅度
   - 表达能力初步判断

4. **经验和潜力**
   - 相关经验（如有）
   - 学习意愿和可塑性

### 👥 面试官分配

HR审核通过后，需要分配以下面试官：
- **舞蹈导师**：评估舞蹈能力和协调性
- **经纪人**：评估商业价值和潜力
- **化妆师**：评估妆容适配度
- **造型师**：评估造型可塑性

### 🔄 状态变化
- **输入状态**：`pending`（待审核）
- **输出状态**：
  - `rejected`（已拒绝）
  - `interview_scheduled`（已安排面试）

### 📊 数据记录
- ✅ HR审核结果和时间
- ✅ 拒绝原因（如被拒绝）
- ✅ 面试官分配记录（如通过）
- ✅ 面试时间安排

### 👥 涉及角色
- **HR管理员**：主导审核和面试官分配
- **系统**：状态更新和通知发送

### ⏱ 时间周期
- HR审核：1-2个工作日

### 👤 候选人视角

#### 等待期间
- **工作台显示**：
  - 申请状态："HR审核中" (status=pending)
  - 进度条：已完成"报名提交"，当前在"HR审核"
  - 预计反馈时间：1-2个工作日

#### 审核通过后
- **收到通知**：
  - 微信消息："您已通过初审，请按时参加面试"
  - 面试时间和地点
- **工作台更新**：
  - 状态变更："已安排面试" (status=interview_scheduled)
  - 显示面试时间、地点、联系人

#### 审核拒绝后
- **收到通知**：
  - 微信消息："感谢您的应聘"
  - 拒绝原因（可选）
- **工作台更新**：
  - 状态变更："未通过审核" (status=rejected)
  - 显示反馈信息（如有）

## 阶段3：线上测试

### 🎯 阶段目标
候选人来公司完成线上性格测试，为后续面试评估提供参考数据。

### 📋 流程详解

```mermaid
sequenceDiagram
    participant C as 候选人
    participant MP as 小程序
    participant S as 系统
    participant DB as 数据库

    C->>C: 1. 按约定时间到达公司
    C->>MP: 2. 打开小程序，进入测试页面
    MP->>S: 3. 请求性格测试题目
    S->>MP: 4. 返回测试题（约50题）
    C->>MP: 5. 完成答题（20-30分钟）
    MP->>S: 6. 提交测试结果
    S->>S: 7. 计算性格得分
    S->>DB: 8. 保存测试结果
    S->>DB: 9. 更新status=online_test_completed
    S->>MP: 10. 显示完成提示
    MP->>C: 11. "测试完成，请等待面试"
```

### 📝 测试内容

#### 性格测试（16PF人格测试）
- **测试时长**：20-30分钟
- **题目数量**：约50题
- **测试维度**：
  - 乐群性、聪慧性、稳定性、恃强性
  - 兴奋性、有恒性、敢为性、敏感性
  - 怀疑性、幻想性、世故性、忧虑性
  - 实验性、独立性、自律性、紧张性

#### 测试结果
系统自动生成性格特质报告，包括：
- 性格类型（16种人格类型之一）
- 各维度得分
- 适合的主播类型建议
- 潜在优势和风险提示

### 🔄 状态变化
- **输入状态**：`interview_scheduled`（已安排面试）
- **输出状态**：`online_test_completed`（线上测试完成）

### 📊 数据记录
- ✅ 保存性格测试结果
- ✅ 生成人格特质报告
- ✅ 更新候选人状态
- ✅ 记录测试完成时间

### 👥 涉及角色
- **候选人**：完成线上测试
- **系统**：提供题目、计算结果、保存数据

### ⏱ 时间周期
- 测试时长：20-30分钟
- 即时完成，无需等待审核

### 👤 候选人视角

#### 操作步骤
1. **到达公司**：按HR通知的时间到达
2. **打开小程序**：使用已注册的账号登录
3. **进入测试页面**：点击"开始测试"
4. **完成答题**：逐题作答，可随时保存进度
5. **提交测试**：完成所有题目后提交

#### 看到的界面

## 阶段5：系统评级

### 🎯 阶段目标
系统根据舞蹈导师评价和妆容AI分析，自动生成候选人评级和建议薪资，HR和经纪人确认结果。

### 📋 流程详解

```mermaid
sequenceDiagram
    participant S as 系统
    participant AI as 妆容AI
    participant DB as 数据库
    participant HR as HR管理员
    participant A as 经纪人

    S->>DB: 1. 检测status=pending_rating
    S->>DB: 2. 获取舞蹈导师评价
    S->>DB: 3. 获取妆前妆后照片
    S->>AI: 4. 调用妆容AI分析
    AI->>AI: 5. 分析妆容适配度
    AI->>S: 6. 返回AI评分
    S->>S: 7. 综合计算评级
    Note over S: 舞蹈评价(60%) + 妆容AI(40%)
    S->>S: 8. 生成评级等级(A/B/C/D)
    S->>S: 9. 计算建议薪资范围
    S->>DB: 10. 保存评级结果
    S->>HR: 11. 通知HR审核
    S->>A: 12. 通知经纪人审核

    HR->>S: 13. 查看评级报告
    A->>S: 14. 查看评级报告
    HR->>A: 15. 讨论确认

    alt 确认通过
        HR->>S: 16a. 确认评级
        S->>DB: 17a. 更新status=rated
        S->>A: 18a. 通知经纪人准备签约
    else 评级不合格
        HR->>S: 16b. 标记拒绝
        S->>DB: 17b. 更新status=rejected
        S->>C: 18b. 发送拒绝通知
    end
```

### 📊 评级算法

#### 评分计算（100分制）
- **舞蹈导师评分**（60分）
  - 基础功底 × 15%
  - 节奏感 × 10%
  - 身体协调性 × 15%
  - 表现力 × 10%
  - 可塑性 × 10%

- **妆容AI评分**（40分）
  - 妆前妆后对比度 × 15%
  - 五官适配度 × 15%
  - 镜头表现力 × 10%

#### 评级等级划分
- **A级**（85-100分）：优秀
  - 建议薪资：8000-12000元/月
  - 培训周期：2周
  - 预期表现：快速上手，高潜力

- **B级**（70-84分）：良好
  - 建议薪资：6000-8000元/月
  - 培训周期：3周
  - 预期表现：正常培养，稳定发展

- **C级**（60-69分）：合格
  - 建议薪资：5000-6000元/月
  - 培训周期：4周
  - 预期表现：需加强培训，潜力待发掘

- **D级**（<60分）：不合格
  - 建议：拒绝录用

### 🔄 状态变化
- **输入状态**：`pending_rating`（待评级）
- **输出状态**：
  - `rated`（已评级，通过）
  - `rejected`（已拒绝，不通过）

### 📊 数据记录
- ✅ 评级结果（A/B/C/D）
- ✅ 综合得分（1-100）
- ✅ 舞蹈评分明细
- ✅ 妆容AI评分明细
- ✅ 建议薪资范围
- ✅ 建议培训周期
- ✅ HR确认记录

### 👥 涉及角色
- **系统**：自动计算评级
- **妆容AI**：分析妆容适配度
- **HR管理员**：审核评级结果
- **经纪人**：参与确认决策

### ⏱ 时间周期
- 系统评级：实时自动完成
- HR确认：1个工作日

### 👤 HR/经纪人视角

#### 评级报告内容
- **候选人基本信息**：姓名、年龄、照片
- **评级结果**：
  - 等级：A级
  - 综合得分：88分
  - 舞蹈评分：52/60分

#### 操作选项

### 👤 候选人视角

#### 智能分账

**计算公式**

```
主播实际收入 = 基础薪资 + 流量奖励 + 打赏分成 + 特殊奖励 - 扣款项
```

**1. 基础薪资计算**
```
底薪：根据合同约定
全勤奖：直播天数≥22天 → 全勤奖
时长补贴：超过基础时长 × 单价
绩效奖金：达成目标 → 对应档位奖金
```

**2. 流量奖励计算**
```
观看时长奖励：根据累计观看时长阶梯
新增粉丝奖励：根据月新增粉丝阶梯
留存率奖励：次日留存≥30% 且 7日留存≥20%
互动奖励：平均互动率≥5%
```

**3. 打赏分成计算**
```
按阶梯分成比例计算
月打赏收入 × 对应档位分成比例

特殊礼物加成：特定礼物 +5%
活动倍率：活动期间 ×1.2
团队合作：联动直播 +10%
```

**4. 特殊奖励**
```
活动排名奖金：根据排名发放
内容创新奖：爆款内容奖励
进步奖励：数据增长达标奖励
师徒奖励：带新人奖励
```

**5. 扣款项**
```
违约扣款：迟到、早退、缺勤
质量扣款：直播质量问题
违规扣款：违反平台规则
其他扣款：借款、赔偿等
```

**税务计算辅助**
```
月收入≤5000元：免税
月收入5000-8000元：3%税率
月收入8000-17000元：10%税率
月收入≥17000元：20%税率
（实际税率以国家政策为准）
```

```mermaid
    D --> E[与经纪人沟通]
    E --> F{核实结果}
    F -->|需修正| G[修正数据]
    F -->|无需修正| H[添加备注说明]
    G --> C
    H --> C
    C --> I{经纪人确认}
    I -->|确认无误| J{金额判断}
    I -->|有异议| E
    J -->|<5万| K[财务审核通过]
    J -->|≥5万| L[提交超管审批]
    L --> M{超管审批}
    M -->|通过| K
    M -->|驳回| N[重新核查]
    N --> B
    K --> O[发放收益]
```

## 关键决策点汇总

### 决策点总览

| 阶段 | 决策点 | 决策者 | 决策标准 | 影响 |
|------|--------|--------|---------|------|
| 筛选 | 通过/拒绝/待定 | HR管理员 | AI匹配度+人工判断 | 是否进入面试 |
| 面试 | 录用/拒绝 | HR+经纪人 | 综合评分≥70分 | 是否签约 |
| 签约 | 合同类型选择 | HR+经纪人 | 主播经验和潜力 | 收益模式 |
| 签约 | 收益方案配置 | 经纪人+财务 | 评估结果和成本 | 主播收入 |
| 培训 | 考核通过/继续培训 | 经纪人 | 考核≥80分 | 是否开播 |
| 运营 | 排班调整 | 运营+经纪人 | 数据表现和档期 | 曝光和收益 |
| 运营 | 问题干预 | 运营专员 | 实时监控预警 | 直播质量 |
| 结算 | 异常数据处理 | 财务+经纪人 | 数据核实结果 | 收益准确性 |
| 结算 | 大额审批 | 超级管理员 | 金额≥5万 | 资金安全 |
