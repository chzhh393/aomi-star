### ✅ 问题2: 星探邀请成功后,绑定关系未生效

**根本原因**: 招募首页没有接收和传递星探推荐码

**完整流程断裂点**:
```
星探分享 → 招募首页(/pages/recruit/index?ref=ABC123)
                  ↓
                ❌ 首页没有保存 ref 参数
                  ↓
          跳转报名页(/pages/recruit/apply)
                  ↓
                ❌ 没有带 ref 参数
                  ↓
          报名提交时 scoutShareCode 为空
                  ↓
                绑定失败 ❌
```

**修复内容**:

#### 1. 修改 `miniprogram/pages/recruit/index/index.js`

**修改点1**: 添加 `scoutShareCode` 数据字段
```javascript
data: {
  // 星探推荐码
  scoutShareCode: '',
  // ... 其他数据
}
```

**修改点2**: 在 `onLoad` 中接收推荐码
```javascript
onLoad(options) {
  // 获取星探推荐码
  const { ref } = options;
  if (ref) {
    this.setData({ scoutShareCode: ref });
    console.log('[招募首页] 检测到星探推荐码:', ref);
  }
  // ...
}
```

**修改点3**: 在 `goToApply()` 中传递推荐码
```javascript
// 传递星探推荐码
const scoutShareCode = this.data.scoutShareCode;
const url = scoutShareCode
  ? `/pages/recruit/apply/apply?ref=${scoutShareCode}`
  : '/pages/recruit/apply/apply';
wx.navigateTo({ url });
```

**修改点4**: 在 `checkStatus()` 中也传递推荐码
```javascript
const scoutShareCode = this.data.scoutShareCode;
const url = scoutShareCode
  ? `/pages/recruit/apply/apply?ref=${scoutShareCode}`
  : '/pages/recruit/apply/apply';

```

#### 2. 修改 `admin-web/src/views/candidates/list.vue`

## 修复后的完整流程

### 星探推荐绑定流程
```
1. 星探分享小程序
   ↓ path: /pages/recruit/index/index?ref=ABC123

2. 候选人点击进入招募首页
   ↓ onLoad(options) 接收 ref 参数
   ↓ 保存到 data.scoutShareCode

3. 点击"立即报名"按钮
   ↓ goToApply() 获取 scoutShareCode
   ↓ 跳转到 /pages/recruit/apply/apply?ref=ABC123

4. 报名页面
   ↓ onLoad(options) 接收 ref 参数
   ↓ 保存到 data.scoutShareCode

5. 填写表单并提交
   ↓ submitApply() 包含 scoutShareCode
   ↓ 调用云函数 candidate.submit

6. 云函数处理
   ↓ 根据 scoutShareCode 查询星探信息
   ↓ 创建 referral 绑定关系
   ↓ 更新星探统计数据

7. ✅ 绑定成功!
```

### 管理后台显示流程
```
1. 候选人列表页面
   ↓ 显示星探推荐标签(如果有 referral 字段)

2. 候选人详情页面
   ↓ 显示星探推荐信息块
   ↓ - 星探姓名
   ↓ - 推荐码
   ↓ - 推荐时间
```

### 星探后台显示流程
```
1. 星探工作台(/pages/scout/home)
   ↓ 调用云函数 scout.getMyReferrals
   ↓ 查询条件: referral.scoutId = 当前星探ID

2. 显示推荐列表
   ↓ 候选人姓名、状态、报名时间
   ↓ 按状态筛选(待审核/已通过/已拒绝/已签约)
```

## 注意事项

1. **推荐码必须正确传递**
   - 星探分享时必须包含 `ref` 参数
   - 招募首页必须保存并传递推荐码
   - 报名页面必须接收推荐码

2. **数据库一致性**
   - 候选人记录中的 `referral.scoutId` 必须对应真实的星探
   - 星探统计数据需要实时更新

3. **权限和结算**
   - 星探只能查看自己推荐的候选人
   - 管理员可以查看所有推荐关系
   - 结算时需要根据 `referral` 字段计算佣金

4. **审核通知**
   - 用户需要在报名时授权订阅消息
   - 如果用户拒绝授权,无法收到通知
   - 建议在UI上添加明显的授权提示
