# 二级星探系统设计方案

## 一、业务场景分析

### 当前系统（一级星探）
```
平台 → 一级星探 → 候选人
```
- 一级星探：直接在平台注册成为星探
- 获得推荐码（6位字母数字）
- 推荐候选人报名

### 目标系统（多级星探）
```
平台 → 一级星探 → 二级星探 → 候选人
              ↓
           候选人（一级星探也可以直接推荐候选人）
```

---

## 二、核心设计要点

### 1. 层级关系
- **一级星探（L1）**：直接从平台注册，没有上级
- **二级星探（L2）**：通过一级星探的邀请码注册
- **可选：三级星探（L3）**：通过二级星探的邀请码注册（建议限制最多3级）

### 2. 权限与功能
所有层级星探的基本功能相同：
- ✅ 推荐候选人
- ✅ 查看推荐列表
- ✅ 查看推荐统计
- ✅ 邀请下级星探（获得邀请码）
- ✅ 查看团队树状结构

### 3. 收益分配（示例）
```
候选人签约 → 总佣金 1000元

情况1：一级星探直接推荐候选人
- 一级星探：1000元（100%）

情况2：二级星探推荐候选人
- 二级星探：700元（70%）- 直接推荐奖励
- 一级星探（上级）：300元（30%）- 团队管理奖励

情况3：三级星探推荐候选人
- 三级星探：600元（60%）
- 二级星探（上级）：250元（25%）
- 一级星探（最高上级）：150元（15%）
```

## 七、防作弊机制

### 1. 层级限制
```javascript
// 最多3级
if (parentScout.level.depth >= 3) {
  return {
    success: false,
    error: '已达到最大层级限制（3级）'
  };
}
```

### 2. 循环检测
```javascript
// 防止A邀请B，B又邀请A的情况
function checkCircularReference(newScoutOpenId, parentScoutId) {
  // 检查新星探是否在父级链条中
  let currentScout = parentScout;
  while (currentScout) {
    if (currentScout.openId === newScoutOpenId) {
      return true; // 检测到循环
    }
    currentScout = getParentScout(currentScout.level.parentScoutId);
  }
  return false;
}
```

### 3. 邀请码有效期
```javascript
{
  inviteCode: 'INV456',

  inviteCodeMaxUse: null   // null表示无限制，或设置最大使用次数
}
```

### 4. 重复注册检查

## 九、关键代码示例

### 注册时处理邀请码

```javascript
// cloudfunctions/scout/index.js

async function registerScout(openId, data) {
  const { inviteCode } = data;

  let parentScout = null;
  let level = { depth: 1, parentScoutId: null };

  // 如果提供了邀请码
  if (inviteCode) {
    // 验证邀请码
    const parentRes = await db.collection('scouts').where({
      inviteCode: inviteCode,
      status: 'active'
    }).get();

    if (parentRes.data.length === 0) {
      return { success: false, error: '邀请码无效或已失效' };
    }

    parentScout = parentRes.data[0];

    // 检查层级限制
    if (parentScout.level.depth >= 3) {
      return { success: false, error: '已达到最大层级限制' };
    }

    // 设置层级信息
    level = {
      depth: parentScout.level.depth + 1,
      parentScoutId: parentScout._id,
      parentScoutName: parentScout.profile.name,
      parentInviteCode: inviteCode
    };
  }

  // 生成邀请码和推荐码
  const shareCode = generateShareCode();
  const newInviteCode = generateInviteCode();

  // 创建星探记录
  const scoutData = {
    openId,
    profile: { ... },
    shareCode,       // 推荐候选人用
    inviteCode: newInviteCode,  // 邀请下级星探用
    level,
    team: {
      directScouts: 0,
      totalScouts: 0,
      activeScouts: 0
    },
    stats: { ... },
    status: 'active',
    createdAt: db.serverDate(),
    updatedAt: db.serverDate()
  };

  const createRes = await db.collection('scouts').add({ data: scoutData });

  // 如果有上级，更新上级的团队统计
  if (parentScout) {
    await updateParentTeamStats(parentScout._id);
  }

  return {
    success: true,
    scoutId: createRes._id,
    shareCode,
    inviteCode: newInviteCode,
    level
  };
}

// 更新上级星探的团队统计
async function updateParentTeamStats(parentScoutId) {
  await db.collection('scouts').doc(parentScoutId).update({
    data: {
      'team.directScouts': _.inc(1),
      'team.totalScouts': _.inc(1),
      updatedAt: db.serverDate()
    }
  });

  // 递归更新更高层级的totalScouts
  const parentScout = await db.collection('scouts').doc(parentScoutId).get();
  if (parentScout.data.level.parentScoutId) {
    await db.collection('scouts').doc(parentScout.data.level.parentScoutId).update({
      data: {
        'team.totalScouts': _.inc(1),
        updatedAt: db.serverDate()
      }
    });
  }
}
```

## 十、总结

### 优势
1. **灵活性**：支持多级推荐，扩展性强
2. **激励性**：团队奖励机制，鼓励发展下级
3. **可追溯**：完整的推荐链条，便于管理和分佣
4. **防作弊**：层级限制、循环检测等机制

### 注意事项
1. **层级限制**：建议最多3级，避免过度复杂
2. **收益透明**：明确告知各级分佣比例
3. **数据一致性**：团队统计需要保持准确
4. **性能优化**：树状结构查询需要优化

### 后续扩展
- 星探等级制度（铜牌/银牌/金牌）
- 团队PK活动
- 收益提现功能
