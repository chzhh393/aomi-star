# 星探分账方案

## 📊 分账规则（方案1 - 70/30）

### 场景1：一级星探直接推荐
```
候选人签约 → 奖励100%给一级星探
```

**示例**：
- 签约奖励：1000元
- 一级星探：1000元（100%）

---

### 场景2：二级星探推荐
```
候选人签约 → 二级星探70% + 一级星探30%
```

**示例**：
- 签约奖励：1000元
- 二级星探（直接推荐人）：700元（70%）
- 一级星探（团队管理）：300元（30%）

**比例说明**：
- **70%** - 二级星探（直接推荐人，主要贡献者）
- **30%** - 一级星探（团队管理、培训支持）

---

```javascript
const COMMISSION_CONFIG = {
```

## 🔄 分账流程

```
候选人签约（status: signed）
    ↓
自动触发分账计算
    ↓
根据推荐链条计算分账
    ↓
更新候选人commission字段
    ↓
更新星探earnings字段
    ↓
管理员确认支付
    ↓
更新状态为已支付
```

## 📋 数据结构

### 1. 候选人数据（candidates）
```javascript
{
  _id: "candidate_id",
  status: "signed",
  referral: {
    scoutChain: ["一级ID", "二级ID"],
    scoutChainNames: ["张三", "李四"]

  // 分账信息

    status: "calculated",      // pending/calculated/paid
    totalAmount: 1000,         // 总奖励金额
    calculatedAt: Date,        // 计算时间
    paidAt: Date,              // 支付时间
    
    // 分账明细
    distribution: [
      {
        scoutId: "二级星探ID",
        scoutName: "李四",
        level: 2,
        amount: 700,
        percentage: 70,
        type: "direct",        // direct/team
        status: "pending"
      },
      {
        scoutId: "一级星探ID",
        scoutName: "张三",
        level: 1,
        amount: 300,
        percentage: 30,
        type: "team",
        status: "pending"
      }
    ]
  }
}
```

```javascript
{
  _id: "scout_id",
```

## 🎯 使用流程

### 1. 候选人签约
管理员在候选人管理页面，将候选人状态更新为"已签约"

### 2. 自动计算分账
系统自动：
1. 识别推荐链条
2. 计算分账金额
3. 更新候选人commission字段
4. 更新星探earnings字段

### 3. 管理员确认支付
1. 进入"分账管理"页面
2. 查看待支付列表
3. 核对分账明细
4. 点击"确认支付"按钮
5. 系统更新状态为已支付

## 💡 注意事项

1. **数据安全**
   - 分账金额需要财务审核
   - 避免重复计算分账
   - 记录完整的分账历史

2. **性能优化**
   - 大量分账时考虑批量处理
   - 定期归档已支付记录
   - 优化查询性能

3. **业务规则**
   - 只有已签约候选人才触发分账
   - 分账状态流转：pending → calculated → paid
   - 支付确认后不可撤销

4. **扩展性**
   - 预留多档位奖励字段
   - 预留提现相关字段
   - 支持分账规则配置
